Hello and thank you for using NordVader Library!

Is a toolkit for Unreal Engine developers, offering asynchronous functions to streamline workflows. It handles complex, resource-intensive tasks without disrupting the main game thread, ensuring smooth and effective performance.

Version 0.03

New features
  • UMemoryWidget
    • Description

      A UI widget that displays real-time memory usage statistics within the game interface.

    • Parameters:
      • None
    • Functionality:
      • Periodically updates and displays total and used physical and virtual memory, as well as their peak values. Helps developers and users monitor system performance and manage resource usage effectively.
    • Usage:
      • Create your own BP widget inherited from MemoryWidget.
      • Add text blocks named TotalPhysicalText, UsedPhysicalText, PeakUsedPhysicalText, TotalVirtualText, UsedVirtualText, and PeakUsedVirtualText and set them as variables.
      • The widget will automatically update these text blocks with the current memory usage statistics.
  • Update Logic for Informational Windows
    • Description: Added new update logic for displaying informational windows.
    • Functionality: Users can now configure how often the update window appears when opening the project.
    • Benefits:
      • Enhanced Control: Users can choose to display the update window every time the project is opened or only when the plugin version changes.
      • Improved User Experience: Provides flexibility and prevents unnecessary interruptions for users who prefer not to see the update window frequently.
    • Project Settings Control:
      • Location: The plugin settings can be managed through the Project Settings menu in Unreal Engine under Plugins -> NordVaderLibrary.
      • Available Setting:
        • Show Always: A boolean setting that determines whether the update window appears every time the project is opened or only when the plugin version changes.
Fixes
  • Fixed function memory leaks, when exiting the simulation, the world is destroyed, and the delegate is still trying to work out what caused the crash.
Added Functions:

Actor Management:

  • AsyncDestroyActorTask: Asynchronously destroys an actor and calls a delegate upon completion.
  • AsyncPlayAnimationTask: Asynchronously plays an animation on a skeletal mesh component.
  • AsyncSetAnimInstanceClassTask: Asynchronously sets the animation instance class for a skeletal mesh component.

Parallel Processing:

  • CheckActorsInFOVParallel: Asynchronously checks if actors are within a specified Field of View (FOV) angle of the camera.
  • TraceSocketsInSkeletalMeshesByTagParallel: Performs parallel line trace operations from a specified starting location to the socket locations of skeletal mesh components tagged with a specific tag.

Damage Management:

  • ApplyDamageToActors: Applies specified damage to an array of actor-damage structures.

Utilities:

  • GetBoolByChance: Determines if an event should occur based on a given chance.
  • GeneratePointsOnSphereFibonacciParallel: Generates points on a sphere using the Fibonacci lattice method in parallel.
  • PerformFibonacciLatticeTracesParallel: Performs ray traces from the center in directions based on points generated by the Fibonacci lattice method and returns an array of hit results.
  • GetMemoryUsage: Retrieves the current memory usage statistics of the system.

Version 0.01

Added Functions:

Actor Management:

  • SetSeparationWeight: Sets the separation weight for CrowdFollowingComponent.
  • AsyncSpawnActor: Asynchronously spawns an actor in the game world.
  • AsyncSpawnActorTask: Schedules asynchronous actor spawning as a task.
  • OnAsyncSpawnActorComplete: Internal function called upon completion of asynchronous actor spawning.

Emitter and System Management:

  • AsyncSpawnEmiter: Asynchronously spawns a particle emitter.
  • AsyncSpawnEmiterTask: Schedules asynchronous particle emitter spawning as a task.
  • OnAsyncSpawnEmiterComplete: Internal function called upon completion of asynchronous particle emitter spawning.
  • AsyncSpawnSystemAttached: Asynchronously spawns and attaches a Niagara system.
  • AsyncSpawnSystemAttachedTask: Schedules asynchronous spawning and attachment of a Niagara system as a task.
  • OnAsyncSpawnSystemAttachedComplete: Internal function called upon completion of asynchronous Niagara system spawning and attachment.

Decal Management:

  • AsyncSpawnDecalAtLocation: Asynchronously spawns a decal at a specified location.
  • AsyncSpawnDecalAtLocationTask: Schedules asynchronous decal spawning at a specified location as a task.
  • OnAsyncSpawnDecalAtLocationComplete: Internal function called upon completion of asynchronous decal spawning.

Sound Management:

  • AsyncPlaySoundAtLocation: Asynchronously plays a sound at a specified location.
  • AsyncPlaySoundAtLocationTask: Schedules asynchronous sound playback at a specified location as a task.
  • OnAsyncPlaySoundAtLocationComplete: Internal function called upon completion of asynchronous sound playback.

Parallel Processing:

  • AsyncFor: Executes a parallel for loop asynchronously.
  • GetActorsBySoftClassParallel: Asynchronously filters actors by class using parallel processing.
  • GetWasRecentlyRenderedbyActorsParallel: Asynchronously checks if actors were recently rendered using parallel processing.
  • ParallelSortActorsByDistance: Asynchronously sorts actors by distance to a point using multi-threading.

Utilities:

  • CreateSubArrayActors: Creates a subarray from a given array of actors.
  • MergeActorArrays: Merges multiple arrays of actors into a single array.
  • MergeSortedActorArraysbylocationToNearest: Merges sorted arrays of actors by their proximity to a specific location.
  • GetActorsBySoftClass: Filters actors by a specified soft class.
  • SortActorsByDistance: Sorts actors by their distance to a specified location.
  • GetWasRecentlyRenderedbyActors: Filters actors that were recently rendered.
  • InRange: Checks if a value is within a specified range.
  • InRange_Pure: Pure version of the range check function.
  • RandomFloatInRange: Generates a random float within a specified range.
  • RandomFloatInRange_Pure: Pure version of the random float generation function.

Damage Management:

  • ApplyDamageSafely: Safely applies damage to an actor using a critical section for thread safety.
  • ApplyDamageActorsTask: Asynchronously applies damage to multiple actors as a task.
  • ApplyDamageActorTask: Asynchronously applies damage to a single actor as a task.
  • ApplyPointDamageActorTask: Asynchronously applies point damage to an actor.
  • ApplyRadialDamageActorTask: Asynchronously applies radial damage.
