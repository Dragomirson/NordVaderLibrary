1. Установка

Для начала сгенерируйте файлы Visual Studio .sln

2. Начало работы

В папке Main/Blueprints/Nodes вы найдёте файл основной истории - MainStory. А также ноды элементов истории. В этом графе вы будете писать вашу историю.

При создании истории необходимо начинать с главы, которая ОБЯЗАТЕЛЬНО должна называться RootChapter!

Описание основных классов построения квестовой логики проекта

NODEНазначение
ChapterNodeЭто глава, которая включает в себя описание, информацию отображения и стиль карточки этой главы.
EventNodeЭто событие, которое содержит визуальную и описательную информацию. А также возможность включения мини-игр. Имеет условную активацию.
CardNodeПередняя сторона карточки действия, имеет условное отображение.
CardResultNodeРезультат совершения действия, так же содержит информацию об отображении. Применяет к игроку теги (предметы, навыки, достижения)
GameEndNodeКонец игры, используется для показа итогов игры и так же имеет визуальную составляющую.

В каждом классе есть раздел BP Graph Node Template, который позволяет настроить фон и цвет отображения на графе.

Background Color 1Настраивает цвет рамки
Background Color 2Цвет фона. Если установлено изображение будет влиять на её цвет.
Для исходного вида отображения картинки используйте белый цвет.
IconФоновая картинка блока

Таблица структуры BP Graph Node Template

3. Теги

В проекте используется система тегов, для хранения информации об инвентаре игрока, о его достижениях и навыках, получаемых в результате действий.

Которая находится в папке

Main/Tables

в файле

TagViewsTable.

TagМетка данных о предмете/достижениях/навыках
NameОтображаемое имя объекта (предмета, достижения, навыка)
TooltipTextПодсказка о предмете при наведении на него
PictureИзображение объекта используемое для его отображения
MaxCountМаксимально возможное кол-во элементов объекта.

Отображение имеющихся тегов происходит в виджете UserScore в папке: Main/Blueprints/Widgets
